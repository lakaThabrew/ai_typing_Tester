## ğŸš€ Getting Started

### Prerequisites

- **Node.js** (v18 or higher)
- **MongoDB** (local or MongoDB Atlas)
- **Gemini API Key** (from Google AI Studio)

### Installation

1. **Clone the repository**

   ```bash
   git clone https://github.com/lakaThabrew/ai-typing-tester.git
   cd ai-typing-tester
   ```

2. **Install dependencies**

   ```bash
   npm install
   ```

3. **Configure environment variables**

   Create `backend/.env` file:

   ```env
   # Server Configuration
   PORT=5000
   NODE_ENV=development

   # MongoDB Connection
   MONGODB_URI=mongodb://localhost:27017/typing-tester
   # OR for MongoDB Atlas:
   # MONGODB_URI=mongodb+srv://username:password@cluster.mongodb.net/typing-tester

   # JWT Secret (generate a random string)
   JWT_SECRET=your_secure_random_jwt_secret_key_here

   # Gemini API Key (Google AI)
   GEMINI_API_KEY=your_gemini_api_key_here

   # Frontend URL (for CORS)
   FRONTEND_URL=http://localhost:5173
   ```

4. **Get Gemini API Key**
   - Visit [Google AI Studio](https://makersuite.google.com/app/apikey)
   - Click "Create API Key" or "Get API Key"
   - Copy the key and add to `.env`

### Running the Application

#### Development Mode (Both Frontend & Backend)

```bash
npm run dev:all
```

#### Run Frontend Only

```bash
npm run dev
```

Frontend will run on `http://localhost:5173`

#### Run Backend Only

```bash
npm run dev:backend
```

Backend will run on `http://localhost:5000`

#### Production Build

```bash
npm run build
npm run start:backend
```

## ğŸ› Troubleshooting

### Backend won't start

- Verify MongoDB is running (if using local MongoDB)
- Check `.env` file exists and has all required variables
- Ensure `MONGODB_URI` connection string is correct

### Gemini AI not generating phrases

- Verify `GEMINI_API_KEY` is set correctly in `.env`
- Check API key has not exceeded quota
- Check console logs for detailed error messages
- Fallback phrases will be used if AI fails

### Frontend can't connect to backend

- Ensure backend is running on `http://localhost:5000`
- Check CORS settings in backend
- Verify API_URL in frontend matches backend URL

### Authentication issues

- Clear localStorage and try logging in again
- Check JWT_SECRET is set in backend `.env`
- Verify token hasn't expired (1-hour expiry)
